# 📋 작업지시서: 지난 1년 일자별(Daily) 쇼핑 트렌드 수집 및 분석 시스템 구축

본 작업지시서는 `docs/` 내에 저장된 네이버 공식 API 문서(데이터랩 및 쇼핑인사이트)를 기반으로 **특정 키워드/카테고리의 최근 1년 치 '일자별(Date)' 쇼핑 검색 트렌드**를 수집하여 대시보드에 연동하는 과정을 명세합니다.

---

## 1. 업무 개요
*   **목표**: 특정 주력 품목이 아닌, **네이버 쇼핑 대분류/중분류 카테고리별 상위 랭크 TOP 50개 키워드**를 베이스로 삼아 최근 1년(365일) 간 '일자별' 쇼핑 클릭량 추이를 수집하고, 선행 트렌드 및 유망 아이템 리스트를 도출합니다.
*   **사용 API 및 기술**: 
    1. **TOP 50 발굴 (Hybrid)**: 네이버 공식 DataLab API는 "카테고리별 상위 500위" 리스트업 자체를 뿌려주지 않는 한계가 있습니다. 따라서 크롤링(Selenium/BeautifulSoup)을 통한 쇼핑인사이트 웹 브리지 로직, 혹은 API 내 정규 카테고리 클릭률(`/v1/datalab/shopping/categories`)의 하위 분류 조회 기능을 병용합니다.
    2. **일간 트렌드 추적**: 네이버 데이터랩 - **쇼핑인사이트 키워드별 트렌드 조회 API** (`/v1/datalab/shopping/category/keyword`)
*   **핵심 활용 변수**: `timeUnit: "date"` (일간 데이터 추출)

---

## 2. API 요청 (Request) 스펙 설계
공식 문서(`docs/shopping_insight_guide.md`)를 기반으로 아래와 같이 요청 파라미터를 구성합니다.

### 2.1. 필수 파라미터 (POST JSON Body)
*   **`startDate`**: 시작 일자 (예: 현재 날짜로부터 365일 전 `2025-03-05`)
*   **`endDate`**: 종료 일자 (예: 현재 날짜 `2026-03-05`)
*   **`timeUnit`**: 반환 구간 단위. 반드시 **`"date"`** 로 고정 (일자별 분석 목적)
*   **`category`**: 네이버 쇼핑 카테고리 코드 배열 (예: 생활/건강 카테고리)
*   **`keyword`**: (키워드 트렌드일 경우) 검색어 배열

### 2.2. 요청 예제 코드 (Python)
```python
import requests
import json
from datetime import datetime, timedelta

# 시작일, 종료일 산출 (최근 1년)
end_date = datetime.now()
start_date = end_date - timedelta(days=365)

url = "https://openapi.naver.com/v1/datalab/shopping/category/keyword"
headers = {
    "X-Naver-Client-Id": "클라이언트_ID",
    "X-Naver-Client-Secret": "클라이언트_시크릿",
    "Content-Type": "application/json"
}

payload = {
    "startDate": start_date.strftime("%Y-%m-%d"),
    "endDate": end_date.strftime("%Y-%m-%d"),
    "timeUnit": "date",  # 핵심: 일자별 데이터 요청
    "category": "50000008", # 검색하려는 1Depth 카테고리 코드 (예: 패션의류, 생활/건강 등)
    "keyword": [
        # 추출된 '상위 50개' 키워드의 배열 반복 (API 단건 한도 5개에 맞춰 10회 분할 요청 로직 추가 필요)
        {"name": "상위키워드1", "param": ["상위키워드1"]},
        {"name": "상위키워드2", "param": ["상위키워드2"]}
    ],
    "device": "", # 전체 기기
    "gender": "", # 전체 성별
    "ages": []    # 전체 연령
}

response = requests.post(url, headers=headers, json=payload)
print(response.json())
```

## 3. 데이터 파싱 (Response) 및 저장 로직
응답으로 반환되는 JSON 데이터에서 `results[].data` 속성 내의 `period`(일자)와 `ratio`(클릭률 상댓값)를 추출하여 `Pandas DataFrame`으로 파싱합니다.

1.  **결측치 처리**: 365일에 해당하는 데이터 중, 검색량이 0에 수렴하여 `ratio` 객체가 누락된 날짜는 `0`으로 채워 시계열의 연속성을 보장합니다.
2.  **데이터 저장**: 수집된 DataFrame 결과물은 `data/daily_trend_YYYYMMDD.csv` 형식으로 로컬 저장합니다.
3.  **UI 렌더링**: `app.py` 시장 수요 동향 탭을 고도화하여 기존 월간(`month`) 조회에서 사용자가 일간(`date`) 버튼 클릭 시 본 로직이 가동되도록 연결합니다.

## 4. 예외 처리 및 방어 로직 (통합 관제탑 공통)
*   **429 Too Many Requests (호출 한도 방어)**:
    일자별(Date) 데이터는 Payload가 크고 조회가 세분화되어 있어, 호출 빈도가 급증할 수 있습니다. 
    `app.py`에 적용된 **5중 API Key 로테이션(0.5초 대기 스위칭)** 엔진을 반드시 거쳐서 호출해야 합니다. (`safe_api_request()` 모듈 100% 필수 래핑)
*   **400 Bad Request (비정상 파라미터 방어)**: 
    시작 날짜(`startDate`)가 API 허용점(2017년 8월 1일) 이전이거나, 포맷이 `YYYY-MM-DD`가 아닐 경우 에러가 발생합니다.
*   **API Payload 제약 처리**:
    하나의 API 호출 당 `keyword` 그룹은 최대 5개까지만 보낼 수 있습니다. 따라서 **상위 50개 키워드**를 조회하기 위해서는 반드시 5개씩 10번으로 청크(Chunk)를 쪼개서 `safe_api_request()`에 태워 보내는 병합/루프 로직이 필수입니다.
